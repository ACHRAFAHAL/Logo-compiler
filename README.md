# TP2 - Rust Logo to SVG

This project is a small Rust interpreter/compiler for a Logo-like language.
It tokenizes and parses commands, builds an AST, evaluates commands, and compiles turtle movements into SVG paths.

## Features

- Lexer and parser built with `santiago`
- AST-based command representation
- Turtle-style movement commands
- Pen control (`penup` / `pendown`)
- `repeat` loop with bracketed blocks
- SVG generation with `svg_fmt`

## Project Structure

- `src/main.rs`: parser + AST + Logo compiler that writes `drawing.svg`
- `src/bin/test_svg.rs`: standalone SVG example that writes `square.svg`
- `Cargo.toml`: package metadata and dependencies

## Supported Language

Commands currently supported by the parser:

- `forward <number>`
- `backward <number>`
- `left <number>`
- `right <number>`
- `penup`
- `pendown`
- `repeat <number> [ ... ]`

Example input (from `main.rs`):

```text
penup backward 100 pendown repeat 5 [ forward 15 penup forward 15 pendown ]
```

## Requirements

- Rust toolchain (stable)
- Cargo

## Build and Run

From the project root:

```bash
cargo run
```

This runs the main binary and generates `drawing.svg`.

To run the second binary:

```bash
cargo run --bin test_svg
```

This generates `square.svg`.

## Useful Commands

```bash
cargo check
cargo build

```

## Output Files

- `drawing.svg`: generated by `src/main.rs`
- `square.svg`: generated by `src/bin/test_svg.rs`

## Notes

- The current parser accepts positive integer numbers (`[0-9]+`).
- Input is currently hardcoded in `src/main.rs`.
